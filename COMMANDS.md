# RDang — Инструкция по командам

Все команды начинаются с префикса `/rdang` (или алиасов: `/rd`, `/dang`, `/bdang`, `/adang`, `/edang`, `/holydang`).

## Основные команды

| Команда | Описание | Права |
|---------|----------|-------|
| `/rdang menu` | Открыть меню управления данжами | `rdang.menu` |
| `/rdang spawn self` | Заспавнить данж рядом с собой (30–150 блоков) | `rdang.spawn` |
| `/rdang spawn <количество>` | Заспавнить N данжей в случайных локациях текущего мира | `rdang.spawn` |
| `/rdang spawn <количество> <мир>` | Заспавнить N данжей в указанном мире | `rdang.spawn` |
| `/rdang give key <ник> [количество]` | Выдать ключ от данжа игроку | `rdang.give.key` |
| `/rdang give compass <ник> [количество]` | Выдать компас данжа игроку | `rdang.give.compass` |
| `/rdang schem <название_схемы>` | Установить схему (.schem) рядом с собой | `rdang.schem` |
| `/rdang undo <id>` | Удалить данж по ID региона и восстановить ландшафт | `rdang.undo` |
| `/rdang reload` | Перезагрузить все конфиги плагина | `rdang.reload` |
| `/rdang update` | Проверить и скачать обновление с GitHub | `rdang.update` |
| `/rdang migrate` | Мигрировать YML-файлы под текущую версию | `rdang.migrate` |

## Административные команды

| Команда | Описание | Права |
|---------|----------|-------|
| `/rdang admins test` | Зарегистрировать и заполнить лутом шалкер, на который смотришь | `rdang.admins` |
| `/rdang admins remove` | Удалить шалкер из базы (смотреть нужно на шалкер) | `rdang.admins` |
| `/rdang admins loot` | Заполнить шалкер лутом без регистрации | `rdang.admins` |

## Система привилегий

- `rdang.use` — базовое право на использование команды `/rdang`
- `rdang.admin` — включает ВСЕ права (для OP/админов)

## Примеры использования

```
/rdang spawn self
  → Спавнит 1 данж в радиусе 30–150 блоков от вас.

/rdang spawn 5
  → Спавнит 5 данжей в случайных точках текущего мира.

/rdang spawn 3 world_nether
  → Спавнит 3 данжа в Незере.

/rdang give key Truhot 2
  → Выдаёт 2 ключа игроку Truhot.

/rdang undo dang_42
  → Удаляет данж с регионом dang_42 и восстанавливает ландшафт.
```

## Примечания

- Если на сервере установлен **CoreProtect**, данжи не будут спавниться в зонах с постройками игроков.
- Данжи автоматически удаляются через заданное время (`auto.time` в `auto.yml`), если все шалкеры в них открыты и пусты.
- Схемы (.schem) должны находиться в папке `plugins/RDang/schem/`.
